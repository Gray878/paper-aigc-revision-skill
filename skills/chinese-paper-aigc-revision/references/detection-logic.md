# AIGC 检测底层逻辑

用于解释检测原理、误判风险和“为什么这样改”。不要把本文件当作规避检测指南；目标是让论文更真实、可核验、可辩护。

## 检测器通常看什么

- **语言概率**：低困惑度、高概率词序、常见搭配密集、句式选择过于稳。
- **节奏/爆发度**：人类草稿常有局部粗糙、长短变化、插入解释、迟疑和取舍；生成文本常更均匀。
- **概率秩可视化**：GLTR 类方法观察词是否大量落在模型高概率候选中。
- **概率曲率**：DetectGPT 类方法比较原文与扰动文本在模型概率上的差异。
- **分类器**：用人写/机写语料训练监督模型，容易受学科、语言、模型版本、提示词和文本长度影响。
- **水印/来源证明**：有些系统尝试找水印或元数据，但普通复制粘贴文本通常缺少可靠来源链。
- **学术一致性**：中文论文检测还可能结合论点、方法、数据、引用和结论是否形成可信研究链。

## 中文论文常见误判触发点

- 文献综述按“学者认为-研究表明-综上”机械堆叠。
- 摘要、研究意义、结论充满通用套话，缺少对象、时间、材料、方法。
- 大量抽象名词和政策性表达，任何题目都能套用。
- “首先、其次、最后、综上所述”重复出现，段落长度和句式过于均匀。
- 翻译腔或润色腔把真实作者的取舍、犹豫、限制全部抹平。
- 引用只罗列作者，未说明文献与本文问题的关系。
- 方法部分只写“文献研究法、案例分析法”，没有样本、来源、变量、编码或步骤。

## 分数应该怎样解释

把 AIGC 分数视为“需要人工复核的风险提示”：

- 它通常反映文本片段与 AI 生成文本特征的相似度，不等于论文质量，也不等于学术不端结论。
- 不同平台、版本、模型、文本范围、格式处理会导致结果不同。
- 非正文、参考文献、表格、公式、脚注、修订痕迹、OCR 错误都可能影响报告。
- 低分不证明完全合规，高分也不自动证明欺骗；关键是作者能否提供研究过程和文本责任。

## 信号到修订动作

| 风险信号 | 不推荐做法 | 合规修订动作 |
| --- | --- | --- |
| 句子太顺、太泛 | 加错别字或口语废话 | 加入具体对象、材料、研究边界和判断过程 |
| 综述像模板 | 同义词替换 | 按争议、方法、对象或时间线重组文献关系 |
| 方法太空 | 换连接词 | 补样本、来源、纳排标准、变量定义、编码规则 |
| 结论像口号 | 拉长句子 | 回扣数据/案例，区分发现、解释、建议、限制 |
| 引用可疑 | 生成新文献 | 标 `[待核验]`，要求用户提供原文或 DOI/链接 |
| 报告标红集中 | 逐句洗稿 | 先判定是否为非正文/模板段，再做证据链重建 |

## 解释话术

可这样说明：

```text
这段被标高风险，主要不是因为某个词本身，而是因为它由通用判断、模板连接和缺少证据支撑的抽象句组成。合规处理不是“换个说法”，而是补入作者真实研究中的对象、材料、方法选择和限制说明，使读者能看到这段话为什么只能由本文作者得出。
```

避免这样说：

```text
我会把它改到检测不出来。
```

## 使用限制

不要提供检测器绕过技巧、隐藏字符、翻译循环、恶意释义、风格伪装或“人类化绕检”服务。可以提供学术表达恢复、证据补充、引用核验、版本记录和申诉说明。
