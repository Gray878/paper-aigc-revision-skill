# AIGC 检测逻辑与合规应对

用于解释检测原理、误判风险和“为什么这样改”。不要把本文件当作规避检测指南；目标是让论文更真实、可核验、可辩护。

## 先给用户的边界说明

当用户要求“去 AIGC”“降 AI 痕迹”“人类化”“按检测逻辑反向规避”时，先转译目标：

```text
我可以根据报告和检测信号帮你定位误判风险、补强真实研究材料、重建论证链、核验引用，并准备修改记录或 AI 使用披露；不能承诺绕过检测、指定分数，或用隐藏字符、翻译循环、随机错误等方式伪装来源。
```

## 检测器通常看什么

公开资料和研究论文常把检测信号分成几类。不同平台不会公开完整算法，因此不要把任何单一指标说成确定规则。

- **困惑度/语言概率**：AI 文本常更贴近模型高概率续写，低困惑度、常见搭配密集、句式选择过稳；但学术规范文本、模板段、翻译腔也可能出现类似信号。
- **突发度/节奏变化**：人类草稿常有句长起伏、局部粗糙、插入解释、取舍痕迹和论证转折；生成文本常更均匀。不要为了“起伏”添加废话、错别字或故意逻辑跳跃。
- **概率秩可视化**：GLTR 类方法观察词是否大量落在模型高概率候选中，用于辅助人工判断。
- **概率曲率/扰动稳定性**：DetectGPT 类方法比较原文与扰动文本在模型概率上的差异，判断文本是否更像模型自己生成。
- **监督分类器**：用人写/机写语料训练模型，可能受语种、学科、文本长度、提示词、模型版本、过度润色和样本分布影响。
- **水印、来源证明和元数据**：有些系统尝试识别水印或来源链；普通复制粘贴文本通常缺少可靠来源证明。
- **报告与格式伪影**：非正文、参考文献、表格、公式、脚注、OCR 错误、修订痕迹可能影响报告范围。
- **学术证据链**：中文论文场景还要看论点、方法、数据、引用和结论是否形成可信研究链。

## 不要把检测逻辑反向使用

检测器不是固定公式，且平台会迭代。按“低困惑度就加随机词”“太平稳就加语病”“高概率词就替换生僻词”的方式反向操作，常见后果是：

- 破坏术语稳定性和学术表达。
- 引入虚假作者痕迹，无法面对导师、期刊或学校复核。
- 让论文从“可能模板化”变成“确有规避意图”。
- 掩盖真正问题：缺少材料、方法、引用关系和作者判断。

合规策略是：把检测信号当成复核入口，把文本问题还原为论文写作问题，再用真实材料修复。

## 中文论文常见误判触发点

- 摘要、研究意义、结论充满通用套话，缺少对象、时间、材料、方法。
- 文献综述按“学者认为-研究表明-综上”机械堆叠。
- 方法部分只写“文献研究法、案例分析法”，没有样本、来源、变量、编码或步骤。
- 大量抽象名词和政策性表达，任何题目都能套用。
- “首先、其次、最后、综上所述”重复出现，段落长度和句式过于均匀。
- 翻译腔或过度润色把真实作者的取舍、犹豫、限制全部抹平。
- 引用只罗列作者，未说明文献与本文问题的关系。
- 参考文献、脚注、表格说明、公式附近文字或 OCR 结果被错误纳入正文范围。

## 分数应该怎样解释

把 AIGC 分数视为“需要人工复核的风险提示”：

- 它通常反映文本片段与 AI 生成文本特征的相似度，不等于论文质量，也不等于学术不端结论。
- 不同平台、版本、模型、文本范围、格式处理会导致结果不同。
- 低分不证明完全合规，高分也不自动证明欺骗；关键是作者能否提供研究过程和文本责任。
- 对学校、期刊或出版社场景，优先查其最新政策和报告原文说明。

## 信号到合规修订动作

| 风险信号 | 不推荐做法 | 合规修订动作 |
| --- | --- | --- |
| 语言概率过稳 | 替换生僻词、堆同义词 | 加入具体对象、材料、研究边界和判断过程 |
| 节奏过均匀 | 加错别字、废话、随机口语 | 让句段长度服从论证层次：定义短、证据实、分析可展开 |
| 综述像模板 | 逐句洗稿 | 按争议、方法、对象或时间线重组文献关系 |
| 方法太空 | 换连接词 | 补样本、来源、纳排标准、变量定义、编码规则 |
| 结论像口号 | 拉长句子 | 回扣数据/案例，区分发现、解释、建议、限制 |
| 引用可疑 | 生成新文献 | 标 `[待核验]`，要求用户提供原文或 DOI/链接 |
| 报告标红集中 | 逐句改到“检测不出” | 先判定是否为非正文/格式伪影，再做证据链重建 |

## 解释话术

可这样说明：

```text
这段被标高风险，主要不是因为某个词本身，而是因为它由通用判断、模板连接和缺少证据支撑的抽象句组成。合规处理不是“换个说法”，而是补入作者真实研究中的对象、材料、方法选择和限制说明，使读者能看到这段话为什么只能由本文作者得出。
```

避免这样说：

```text
我会把它改到检测不出来。
```

## 使用限制

不要提供检测器绕过技巧、隐藏字符、翻译循环、恶意释义、风格伪装或“人类化绕检”服务。可以提供学术表达恢复、证据补充、引用核验、版本记录、误判说明和 AI 使用披露。
