# 中文 AIGC 检测报告处理指南

用于用户上传知网、维普、万方、Turnitin 或其他 AIGC 报告时。优先采用报告原文术语，不把不同平台的分数直接横向比较。

## 先抽取字段

把报告整理成表格：

```markdown
| 字段 | 报告原文 | 解释 | 处理建议 |
| --- | --- | --- | --- |
| 检测平台/版本 |  | 平台和版本影响阈值 |  |
| 检测时间 |  | 同一文本重测可能不同 |  |
| 送检字符/字数 |  | 判断文本范围 |  |
| 检测字符/有效字符 |  | 排除参考文献等后的基数 |  |
| 疑似 AI 比例/AI 值 |  | 概率性线索 |  |
| 高风险片段 |  | 优先人工复核 |  |
| 一般风险片段 |  | 看是否模板化 |  |
| 未参与检测内容 |  | 可能含脚注、图表、公式等 |  |
```

## 平台提示

### 知网/CNKI 风格报告

- 注意正文范围。公开页面提示非正文内容如标题、公式、图片、表格、参考文献等建议删除，脚注、尾注、特殊字符、图表、公式等可能暂不支持。
- 结果是线索，不是定性依据。公开说明称 AIGC 值与论文质量无关，表示片段存在 AI 生成可能性的概率，且结果仅供参考。
- 处理重点：先核对报告是否只分析正文，再逐段判断模板化、证据不足或方法缺失。

### 万方风格报告

- 常见字段包括送检字符数、检测字符数、疑似 AI 生成文本字符数、疑似 AI 生成文本占比。
- 示例报告把“显著疑似”设为可能性大于 85%，把“一般疑似”设为大于 50% 且小于等于 85%。
- 报告可能写明被标注文本只表示具有 AI 生成文本的部分特征，不确定为 AI 生成，结果仅供参考。

### 维普/VIP 风格报告

- 报告通常按片段展示疑似风险，并可能附有算法自动生成、结果仅供参考的说明。
- 处理重点：不要只追逐标红句，检查整段是否缺少研究对象、方法、引用关系或作者判断。

### Turnitin/英文报告

- Turnitin 的 AI writing report 使用 qualifying text 概念；不同语种模型能力不同。
- 2024 年起，Turnitin 对 1%-19% 的低分区间用 `*%` 等方式处理，以降低误报风险；官方也提醒 AI score 不应作为对学生采取不利行动的唯一依据。
- 处理重点：区分 AI-generated only、AI-paraphrased 等类别；中文论文只在用户确有英文报告时参考。

## 逐段处理流程

1. **截图/报告转文字**：保留平台、时间、总分、字段定义和标红片段。
2. **排除报告伪影**：非正文、参考文献、表格、公式、脚注、OCR 错误、修订/批注模式。
3. **片段归因**：每段标注 `report artifact`、`generic claim`、`template structure`、`weak evidence` 等。
4. **材料需求**：每个高风险段列出用户必须提供的真实材料。
5. **重写优先级**：显著疑似且证据薄弱的段落优先；一般疑似但有真实材料的段落可轻修。
6. **复核说明**：说明修订依据，而不是说“已降到安全”。

## 常见报告解读话术

```text
报告显示该段为高风险，但报告本身只提供概率性线索。该段确实存在“模板结构 + 弱证据”的问题：它没有说明本文研究对象、材料来源和分析依据。建议不是简单换词，而是补入作者真实研究材料后重写论证链。
```

```text
该标红可能与报告范围有关：片段属于参考文献/表格说明/公式附近文本。建议先确认平台是否把非正文纳入检测，再决定是否修改正文。
```

## 不要这样处理

- 不按报告字段直接宣布“AI 率就是论文 AI 含量”。
- 不把不同平台的百分比当同一指标比较。
- 不逐句机械洗稿。
- 不给“安全阈值”“保证通过”。
