# 中文论文 AIGC 合规修订流程

用于实际改论文。先诊断，再补材料，再写修订稿。

## 输入清单

只索取当前任务必要信息：

- 论文题目、学科、层次：课程论文、本科/硕博论文、期刊稿、会议稿。
- 学校/期刊对 AI 辅助写作、AIGC 检测、披露的要求。
- 检测报告或标红片段，若有。
- 用户真实材料：提纲、阅读笔记、数据表、访谈记录、实验设置、代码、案例来源、政策文本、导师意见。
- 期望动作：解释报告、段落修订、章节重写、全篇计划、披露说明、申诉材料。

## 诊断标签

- `generic claim`：泛泛判断，没有对象、范围、条件。
- `template structure`：模板化段落或固定连接词。
- `weak evidence`：论点没有数据、案例、引用、方法支撑。
- `citation risk`：引用不存在、未核验、堆砌或与论点关系不清。
- `method gap`：样本、变量、步骤、编码、工具、限制缺失。
- `logic jump`：证据不能推出结论。
- `style uniformity`：句长、语气、段落节奏过于均匀。
- `report artifact`：报告范围、格式、OCR、非正文、表格或参考文献导致的风险。

## 修订顺序

1. **确认报告范围**：正文、参考文献、表格、脚注、标题是否被计入。
2. **定位真实问题**：区分“可能误判”和“文本确实空泛”。
3. **列补充材料**：每个高风险段至少要求一种真实支撑：数据、文献、案例、方法、观察或限制。
4. **重写论证链**：按“问题-材料-分析-限制-结论”或学科更合适的结构重组。
5. **语言收束**：保留学术规范，减少套话、空泛形容词和机械连接。
6. **引用核验**：能查证的写清；不能查证的标 `[待核验]`。
7. **验收记录**：说明修改依据、用户还需补充项和是否需要披露 AI 辅助。

## 安全改写动作

- 缩小范围：加时间、区域、对象、样本、语料、制度背景。
- 锚定证据：绑定具体文献、数据表、访谈、案例、政策条款、实验现象。
- 暴露作者决策：为什么选这个案例、指标、方法、比较对象或理论框架。
- 加入分析摩擦：说明例外、竞争解释、适用边界和局限。
- 重建文献关系：从“作者列表”改为“问题谱系/方法差异/观点争议”。
- 稳定术语：不要为了变化而乱换专业概念。
- 记录修改：保留“修改依据”和“待作者核验”。

## 禁止动作

- 编造参考文献、页码、数据、访谈、实验结果、政策条款。
- 只做同义词替换、扩句、缩句、倒装、翻译再翻译。
- 加错别字、语病、不可见字符、谐音字、随机标点或格式污染。
- 许诺“通过检测”“降低到某比例”“检测不出来”。
- 把用户未提供的研究过程写成事实。

## 修订输出模板

```markdown
### 段落/章节
风险标签：
- 

原文问题：
- 

需要作者补充：
- 

修改策略：
- 

修订稿：

修改依据：
- 

验收：
- [ ] 未新增虚构事实
- [ ] 引用已核验或已标 `[待核验]`
- [ ] 作者研究决策可见
- [ ] 需要披露 AI 辅助时已提示
```

## 披露草稿模板

```text
在本研究的[具体环节]中，作者使用[工具名称及版本]辅助完成[用途：如语言润色、结构梳理、引用格式检查、代码解释等]。使用时间为[日期]，关键提示包括[概述提示]。作者已对生成内容进行核验、修改和取舍，并对论文中的观点、数据、结论和引用承担全部责任。相关过程记录可按要求提供。
```

## 申诉/说明材料

当用户面对高 AIGC 分数但认为存在误判时，帮助准备：

- 检测报告字段解释和高风险段落清单。
- 原始草稿、修改历史、导师反馈、阅读笔记、数据/案例来源。
- 逐段说明：为什么该段看起来模板化，修订后补入了哪些作者材料。
- 引用核验清单：已核验、待核验、删除或替换。
- AI 使用披露：如确有使用，说明用途、范围和人工核验。
