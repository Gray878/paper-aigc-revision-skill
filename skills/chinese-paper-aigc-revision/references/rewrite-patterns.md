# 段落级与章节级改写模板

用于把抽象、模板化、AI 腔文本改成有作者研究痕迹的中文论文表达。每次改写前先确认用户能否提供相应材料。

## 通用改写框架

```text
原句功能 -> 缺失信息 -> 可补材料 -> 新论证结构 -> 修订稿
```

优先补材料，不优先换词。

## 摘要

风险：背景泛、方法空、结果像口号。

结构：

```text
研究背景/问题 -> 研究对象与范围 -> 方法/材料 -> 核心发现 -> 贡献/限制
```

改写提示：

- 加论文对象、样本、时间、数据来源。
- 结果写具体发现，不写“具有重要意义”。
- 限制可简短出现，增强可信度。

## 引言/研究意义

风险：宏大叙事和政策套话。

结构：

```text
现实问题 -> 现有研究或实践缺口 -> 本文切入点 -> 研究问题 -> 预期贡献
```

示例动作：

- 把“随着 X 快速发展”改为“在 [年份/地区/行业] 的 [具体变化] 中，出现了 [具体问题]”。
- 把“具有重要意义”拆成理论意义、方法意义或实践意义，并绑定材料。

## 文献综述

风险：作者堆砌、观点雷同、连接词机械。

结构：

```text
研究主题 A：代表观点 -> 方法/对象差异 -> 对本文启示
研究主题 B：争议或不足 -> 本文如何回应
```

改写动作：

- 按问题、方法、对象或时间线分组，不按“张三认为、李四指出”流水账。
- 每组至少说明：该类研究解决了什么、没解决什么、本文为何继续研究。
- 对未核验文献标 `[待核验]`，不补不存在的文献。

## 方法部分

风险：只有“采用文献研究法/案例分析法”，没有可复现细节。

结构：

```text
研究设计 -> 数据/样本来源 -> 纳入排除标准 -> 分析步骤 -> 可靠性/局限
```

需要作者补充：

- 数据或案例来源、时间范围、数量。
- 变量/指标定义。
- 编码规则、统计方法、工具版本。
- 为什么选这些材料而不是其他材料。

## 案例/实证分析

风险：描述多、分析少、结论跳。

结构：

```text
案例背景 -> 关键事实 -> 分析维度 -> 证据解释 -> 反例/限制 -> 小结
```

改写动作：

- 把案例事实和作者分析分开。
- 每个判断对应一个事实、数据、访谈或引用。
- 写出“为什么这个案例能说明本文问题”。

## 结论

风险：复述摘要、政策口号、建议空泛。

结构：

```text
主要发现 -> 与研究问题对应 -> 理论/实践含义 -> 建议 -> 限制与后续研究
```

改写动作：

- 每条结论回扣前文证据。
- 建议写对象、动作、条件和边界。
- 不把本文没有证明的内容写成结论。

## 政策建议/对策

风险：“加强、完善、推动、提升”堆叠。

结构：

```text
问题来源 -> 责任主体 -> 可执行措施 -> 实施条件 -> 风险控制 -> 评价指标
```

示例：

```text
原句：应进一步完善相关制度，加强监督管理。
修订方向：说明由谁完善什么制度，针对什么风险，用什么指标判断有效。
```

## 高风险通用段示例

原段：

```text
随着人工智能技术的快速发展，其在各行各业中的应用越来越广泛，对社会发展产生了深远影响。因此，研究人工智能的应用具有重要理论意义和现实意义。
```

诊断：

- `generic claim`
- `template structure`
- `weak evidence`

可补材料：

- 具体行业、地区、年份。
- 本文研究对象。
- 真实案例、政策文本、数据或访谈。
- 与已有研究相比的缺口。

修订骨架：

```text
在[具体场景]中，[具体技术/制度变化]导致[具体问题]更加突出。已有研究多从[角度]讨论，但对[本文对象]中的[缺口]解释不足。本文据此选取[材料/案例/数据]，重点回答[研究问题]。
```
